# Daily Random Photo - 랜덤 데일리 미션 카메라 앱

## 🚀 빠른 시작

### 필수 요구사항

- Node.js >= 18.0.0
- npm >= 9.0.0
- React Native CLI
- Android Studio (Android 개발용)
- Xcode (iOS 개발용)

### 설치 및 실행

```bash
# 프로젝트 클론
git clone https://github.com/hhee4455/Daliy-Random-Photo.git
cd Daliy-Random-Photo

# 의존성 설치
npm run install:all

# 환경변수 설정
cp .env.example .env
# .env 파일을 열어서 실제 값들로 수정

# 개발 서버 시작
npm run dev

# 모바일 앱 실행
npm run start:android  # Android
npm run start:ios      # iOS
```

## 1. 프로젝트 개요

- **앱 명 (가칭):** 랜덤 데일리 미션 카메라 앱
- **목적:**  
  매일 전 사용자에게 동일한 주제(미션)를 제시, 사용자가 해당 주제에 맞는 사진을 촬영·업로드 → 공유 → 기록/도전 시스템으로 자연스러운 반복성과 바이럴 효과 유발
- **핵심 컨셉:**  
  간단하지만 강력한 ‘하루 한 번 사진 챌린지’ + 소셜 공유 + 연속 도전 보상 시스템

---

## 2. MVP(최소 기능 제품) 핵심 기능

| 기능             | 설명                                                          | 우선순위 |
| ---------------- | ------------------------------------------------------------- | -------- |
| 카카오 로그인    | 카카오톡 소셜로그인 통해 사용자 인증 및 식별                  | 필수     |
| 오늘의 미션 제공 | 모든 사용자에게 매일 동일한 랜덤 미션 주제 노출               | 필수     |
| 사진 업로드      | 카메라 촬영 또는 갤러리에서 사진 선택 후 미션명과 함께 업로드 | 필수     |
| 사진 SNS 공유    | 업로드 직후 인스타그램, 카카오톡 등 외부 SNS를 통한 사진 공유 | 필수     |

---

## 3. 사용자 플로우

```flowchart TD
A[앱 실행] --> B[카카오 로그인]
B --> C[오늘의 미션 확인]
C --> D[사진 촬영 또는 갤러리 선택]
D --> E[사진 업로드 - 미션명 자동 태깅]
E --> F[사진 바로 SNS 공유]
F --> G[미션 완료 및 도전/보상 준비]
```

- 단순하고 직관적인 4단계 흐름으로 설계
- 로그인 → 미션 확인 → 촬영/업로드 → 공유로 자연스러운 참여 동기 부여

---

## 4. 주요 기능 상세 설명

### 4.1 카카오 로그인

- 소셜 인증으로 간편 가입/로그인 구현
- 유저 식별 및 차후 랭킹, 좋아요, 커뮤니티 기능 확장 대비
- 카카오 개발자센터 앱 등록 및 OAuth 설정 필요
- 개인정보처리방침 및 동의 처리 포함

### 4.2 오늘의 미션

- 미션은 서버에서 사전에 등록된 미션 풀 중 하나를 매일 모든 사용자에게 동일하게 제공
- 미션 예시: “오늘의 노란색 찾기”, “웃고 있는 사람 촬영”
- 메인 화면에 크게 표시되어 사용자의 참여 동기 부여

### 4.3 사진 업로드

- 카메라 촬영 또는 기존 갤러리에서 선택 가능
- 사진 업로드 시 미션명 및 관련 메타데이터 자동 태깅
- 사진에 미션명 워터마크 자동 삽입(지원 가능 시)
- 간단한 코멘트 작성 옵션 추가 가능
- 공개 / 비공개 설정은 MVP 단계에서 선택 사항이나, 후속 확장 대비 기능 설계 권장

### 4.4 사진 공유

- 업로드 완료 직후 SNS 공유 버튼 제공
- 인스타그램, 카카오톡, 메시지 등 다양한 채널 지원
- 공유용 이미지에 미션명, 앱 로고 등 워터마크 포함으로 챌린지 바이럴 효과 극대화

## 6. 백엔드 API 기본 설계

| HTTP Method | Endpoint              | 설명                             |
| ----------- | --------------------- | -------------------------------- |
| POST        | /api/auth/kakao-login | 카카오 로그인, 토큰 교환 및 인증 |
| GET         | /api/mission/today    | 오늘의 미션 조회                 |
| POST        | /api/photo/upload     | 사진 업로드 및 메타정보 저장     |
| GET         | /api/photo/mine       | 내 업로드 사진 목록 조회         |
| POST        | /api/photo/share      | SNS 공유용 이미지 및 텍스트 생성 |

## 8. 개발 및 배포 플랜

### 8.1 개발 환경 세팅

- React Native 앱 초기화 (Expo 고려 가능)
- Node.js(Express or NestJS) API 서버 초기 세팅
- PostgreSQL 등 DB 초기화 및 스키마 설계
- AWS S3 버킷 세팅 및 presigned URL 구성
- 카카오 개발자 콘솔에 앱 등록

### 8.2 기능별 개발 순서

1. 카카오 로그인 API 구현 및 연동 확인
2. 미션 API 및 서버 내 미션 풀 세팅
3. 사진 업로드 API 및 S3 연동 (presigned URL)
4. 프론트엔드 ‘오늘의 미션’ 화면 및 카메라 촬영/업로드 구현
5. 업로드 후 SNS 공유 기능 구현
6. 간단한 내 기록 갤러리 구현

### 8.3 테스트 및 배포

- 로컬 및 모바일 디바이스 내 간단한 유저 시나리오 테스트
- Beta 테스트 및 피드백 수집
- 클라우드 서버, 이미지 저장소, 인증서, 도메인 세팅 및 HTTPS 적용
- 앱 빌드 → TestFlight (iOS), APK 배포 (Android)
- 정식 스토어 등록 준비 (설명, 스크린샷, 정책 문서 등)
